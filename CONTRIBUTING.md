# Contribution Guidelines

Thank you for your interest in contributing to **Cosmos Vault — Semester 5**!  
Follow these steps to keep the repository clean, useful, and standardized.

---

## Workflow

1. **Fork** this repository.
2. **Clone your fork:**
   ```bash
   git clone https://github.com/<your-username>/Cosmos-Vault-IIITDM-Sem5.git
   ```
3. Create a new branch:
    ```bash
   git checkout -b add-new-material
   ```
4. Add or update your files in the correct folder. Avoid spaces in filenames, Use `-` or `_`.
5. Commit your changes:
   ```bash
   git add .
   git commit -m "Added new content for [Subject Name / Week]"
   ```
6. Push and open a Pull Request:
   ```bash
   git push origin add-new-material
   ```
7. Provide a brief description of your additions.

---


## Large Files & External Links

- Files larger than 50 MB → Use Git LFS or upload externally (Google Drive, Dropbox, etc.)
- Include shared links in a `LINKS.md` file under the relevant subject folder.

Example:
```markdown
  # External Resources
  - [Lecture Audio - Week 2](https://drive.google.com/...)
```

---

## Code Submissions

- Keep source code clean and modular.
- Add a `README.md` inside each project folder explaining:
  - Project overview
  = Dependencies / Libraries
  - How to run
 
---

## Formatting Standards

- Use lowercase filenames where possible.
- PDFs preferred for notes; Markdown (.md) for text.
- Only educational resources.
